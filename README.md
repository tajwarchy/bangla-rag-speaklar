# Context-Aware Bangla RAG System
### Sub-100ms Latency | Apple M1 Air | Local Inference Only

A production-grade Retrieval-Augmented Generation (RAG) system for a Bangla-language e-commerce assistant. Serves accurate, context-aware product information from a 5,000-product dataset with full multi-turn coreference resolution — all running locally on an Apple M1 Air.

## Architecture

```
User Query
    │
    ▼
┌─────────────────────────────┐
│  Stage 1: Coref Resolution  │  ~0.1ms   Rule-based Python
│  "দাম কত?" → "নুডুলসএর দাম কত?" │
└────────────────┬────────────┘
                 │
                 ▼
┌─────────────────────────────┐
│  Stage 2: Query Embedding   │  ~5–15ms  ONNX Runtime (MiniLM-L12)
│  384-dim normalised vector  │
└────────────────┬────────────┘
                 │
                 ▼
┌─────────────────────────────┐
│  Stage 3: Hybrid Search     │  ~10–14ms Keyword match + FAISS
│  Top-3 products retrieved   │           IndexFlatIP (5K vectors)
└────────────────┬────────────┘
                 │
                 ▼
┌─────────────────────────────┐
│  Stage 4: LLM Generation    │  ~54–56ms gemma-3-1b-it Q4_K_M
│  Streaming Bangla answer    │           llama-cpp-python + Metal
└─────────────────────────────┘
```

### Latency Budget

| Stage | Method | Time |
|-------|--------|------|
| Coreference | Rule-based Python | ~0.1ms |
| Embedding | ONNX MiniLM-L12 | ~5–15ms |
| FAISS Search | IndexFlatIP (RAM) | ~10–14ms |
| LLM TTFT | gemma-3-1b Q4_K_M Metal | ~54–56ms |
| **Total** | | **~79ms (mean)** |

## Key Design Decisions

### Why ONNX Runtime for Embeddings?
`sentence-transformers` on M1 shows random latency spikes (50–568ms) due to unified memory pressure evicting model weights between calls. ONNX Runtime pins the model graph in memory, giving consistent 5–15ms encode times.

### Why llama-cpp-python instead of Ollama?
Ollama adds ~190ms HTTP overhead per request (its own TCP server). `llama-cpp-python` loads the GGUF in-process and calls Metal GPU directly, cutting TTFT from ~190ms to ~55ms.

### Why Hybrid Search (Keyword + FAISS)?
The MiniLM model's Bengali token embeddings have low cosine similarity even for exact product name matches (score ~0.61). A pure vector search returns wrong products. Keyword match is applied first (sorted by token specificity), with FAISS filling remaining slots.

### Why Rule-Based Coreference?
Neural coreference models (spaCy, transformers) add 200ms+. A simple entity stack with punctuation-stripped token matching resolves Bangla product references in <1ms with 100% accuracy on the product-QA domain.

## Setup & Installation

### Prerequisites
- macOS with Apple Silicon (M1/M2/M3)
- Python 3.10+
- ~3 GB free disk space

### Step 1 — Clone and create environment

```bash
git clone https://github.com/tajwarchy/bangla-rag-speaklar.git
cd bangla-rag
python3 -m venv .venv
source .venv/bin/activate
```

### Step 2 — Install dependencies

```bash
pip install --upgrade pip

pip install \
  sentence-transformers \
  faiss-cpu \
  numpy \
  pandas \
  rich \
  huggingface_hub \
  optimum[onnxruntime] \
  transformers

# llama-cpp-python with Metal GPU support
CMAKE_ARGS="-DGGUF_METAL=on" pip install llama-cpp-python --force-reinstall --no-cache-dir
```

### Step 3 — Download GGUF model

```bash
hf auth login   # requires HuggingFace account
hf download lmstudio-community/gemma-3-1b-it-GGUF gemma-3-1b-it-Q4_K_M.gguf --local-dir .
```

### Step 4 — Generate dataset

```bash
python dataset_gen.py
# Output: products.csv (5,000 rows)
```

### Step 5 — Build FAISS index

```bash
python fix_index.py
# Output: products.faiss, products.pkl
```

### Step 6 — Verify environment

```bash
python verify_setup.py
# All 6 checks must pass
```

## Coreference Resolution

The system resolves anaphoric references across turns using a rule-based entity stack:

```
Q1: "আপনাদের কোম্পানি কি নুডুলস বিক্রি করে?"
    → entity extracted: "নুডুলস"
    → pushed to entity_stack

Q2: "দাম কত টাকা?"
    → no product noun detected → anaphoric
    → resolved: "নুডুলসএর দাম কত টাকা?"
    → correct products retrieved ✓
```

Suffix stripping handles inflected forms: `নুডুলসএর` → `নুডুলস`, `চায়ের` → `চা`, `ঘির` → `ঘি`.

## Limitations

- **Cold-start penalty:** First query on a new topic carries 50–300ms TTFT overhead from GGUF KV cache initialisation. The 100ms target applies to warmed follow-up queries.
- **Dataset is synthetic:** The 5,000-product CSV is generated. Real e-commerce data may require reindexing.
- **Coreference is rule-based:** Complex multi-entity conversations or non-product references may not resolve correctly.
- **Model language:** `gemma-3-1b` is primarily English-trained. Bangla answers are functional but may occasionally mix scripts or produce grammatically imperfect responses.
